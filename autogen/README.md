# AutoGen Agent Workflow

This folder contains a small multi-agent pipeline that generates C# code and NUnit tests.

## File structure

| File | What it does |
|---|---|
| `orchestrator.py` | Main entry point. Creates a .NET solution/project/test project, runs the agents, extracts C# code blocks from chat output, and writes final files. |
| `coding_agent.py` | Defines the coding assistant prompt (`AutoGenCodingAssistant`) that generates compilable C# implementation code. |
| `test_suite_agent.py` | Defines the test assistant prompt (`TestSuiteAssistant`) that generates NUnit test code. |
| `reviewer_agent.py` | Defines a reviewer assistant (`AutoGenCodeReviewer`) for code-quality feedback (currently not active in the chat agent list by default). |
| `README.md` | This documentation. |

## What gets generated when you run

`orchestrator.py` creates (and recreates) output under:

`C:\Users\p_vis\source\repos\generated`

Expected output:

- `AutoGeneratedSolution.sln`
- `AutoGenerated\Program.cs` (generated C# implementation)
- `AutoGeneratedTests\Tests.cs` (generated NUnit tests)

## Prerequisites

- Python 3.10+ (recommended)
- .NET SDK installed (`dotnet` available in PATH)
- Python packages:
  - `autogen`
  - `python-dotenv`
- `DEEPSEEK_API_KEY` set in environment (or in `.env`)

## How to run the agent

From the repository root:

```powershell
cd autogen
python orchestrator.py
```

## Change the prompt/task

Edit `input_problem` near the bottom of `orchestrator.py`, then run again.

Note: every run deletes and recreates the output folder in `generated`.

## Troubleshooting

- `dotnet` is not recognized:
  - Install the .NET SDK, then reopen terminal/IDE and run `dotnet --version`.
- Python import errors for `autogen` or `dotenv`:
  - Install packages with `pip install autogen python-dotenv`.
- `DEEPSEEK_API_KEY` missing or invalid:
  - Add it to environment or `.env`, then rerun.
- HTTP/auth/model errors from DeepSeek:
  - Confirm API key value, model name (`deepseek-chat`), and network connectivity.
- Generated files are missing:
  - Check agent output for missing fenced ` ```csharp ` blocks; extraction depends on those blocks.
- Previous output disappeared:
  - Expected behavior. `setup_project()` deletes and recreates `generated` each run.
